This first post covers how to set up this blog. It is built with the Hugo static site generator and uses the Blowfish theme. As this website’s source code is stored on GitHub, check out the Hugo documentation for details on how to host the site on GitHub Pages. (GitHub’s default static site generator is Jekyll, so extra steps are required.)

I tried to customise the typefaces to use IBM Plex Sans and JetBrains Mono, but could never get the CSS to work so I gave up. This website uses the default settings and themes. I hate CSS.

Hugo and Blowfish theme installation
#

This assumes that a Git repository for GitHub Pages has been created. Follow the steps here to get this prerequisite sorted. Clone the repository and navigate to its root level in the terminal.

The next steps are to install the required tools. I use Homebrew to install the following:

  • Hugo (installation of Go isn’t necessary)
  • Node.js (this isn’t strictly necessary, but having it allows us to use Blowfish’s CLI tool for installation and configuration)

Find the Blowfish documentation for installation here. I use the CLI-based blowfish-tools to install the theme.

> npx blowfish-tools

This shows a splash screen and the following interactive menu:

Create the website folder as a subdirectory of the Git repository. If you enter . for the current folder, the process will be aborted following an error message.

Go through the options and change whatever is desired. Hit the Escape key to escape the interactive menu.

Going back into the interactive menu
#

Use the same command to re-enter the interactive menu:

npx blowfish-tools

There is an important caveat here: be at the level of the website directory when running this command. If you are at any other level, the Blowfish installer will think that you are trying to create a brand new website and you’ll see the same message as that shown above.

You know you’re at the correct level when you see this:

Alert boxes (or callouts)
#

Unfortunately, this theme doesn’t implement callouts in the way that Obsidian does. It’s a bit more backwards and more inconvenient to write.

This is how you create an alert box. You can insert any icon listed on this page for added effect.

And this is what it looks like in code:

{{< alert "circle-info" >}}
This is how you create an _alert_ box. You can insert any icon listed [on this page](https://blowfish.page/samples/icons/) for added effect.
{{< /alert >}}

Shortcodes
#

Shortcodes are a kind of markup syntax.

Hugo has a tonne of additional functionality that is exposed by way of shortcodes. The Blowfish documentation on shortcodes provides a thorough description of what’s available. The alert boxes in the previous section are one such example of shortcodes. The icons shortcode is another.

By the way, Mermaid diagrams are created using shortcodes!
